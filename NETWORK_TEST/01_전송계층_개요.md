# 전송계층 개요

> ⭐ IP는 Host 식별, Port는 Process 식별

---

## TCP/IP 프로토콜 슈트 계층 구조

| 계층 | 번호 | 주요 프로토콜 |
|------|------|--------------|
| 애플리케이션 | 7/6/5 | HTTP, FTP, SMTP, DNS, DHCP, NFS, SNMP, Telnet |
| 전송 | 4 | TCP, UDP |
| 인터넷 | 3 | IP(IPv4/IPv6), ICMP, ARP, RARP, IPsec, IPNAT |
| 네트워크 인터페이스 | 2 | SLIP, PPP, LAN/WAN 하드웨어 드라이버 |

---

## IP 계층의 한계 (전송계층이 필요한 이유)

IP는 컴퓨터(Host)까지만 찾아간다. 그 안의 프로그램은 구분하지 못한다.

**IP가 해결하지 못하는 4가지 문제:**

1. **프로그램 구분 불가** — Chrome인지 Discord인지 알 수 없음
2. **데이터 손실** — 패킷이 사라져도 IP는 복구하지 않음
3. **순서 변경** — 1,2,3,4,5 보냈는데 3,1,5,2,4로 도착 가능
4. **수신 여부 불확인** — 상대방이 받았는지 알 수 없음

---

## 전송계층의 역할

| 역할 | 설명 |
|------|------|
| 프로그램 식별 | Port 번호로 올바른 프로그램에 전달 |
| 신뢰성 제공 | 데이터 손실 시 재전송 |
| 순서 유지 | 순서 번호로 재조립 |
| 흐름 제어 | 수신자 처리 속도에 맞춰 전송 |
| 연결 관리 | 연결 생성 및 종료 |

---

## 핵심 암기

```
IP 주소 → 컴퓨터(Host) 찾기
Port 번호 → 프로그램(Process) 찾기
Socket = IP 주소 + Port 번호 (예: 192.168.0.10:80)
```

---

## 전송계층 대표 프로토콜

- **TCP** : 정확성 우선 (연결형, 신뢰성)
- **UDP** : 속도 우선 (비연결형, 빠름)

---

## 자주 틀리는 부분

❌ IP가 프로그램을 구분한다 → Port 번호가 구분  
❌ TCP와 IP는 같은 계층이다 → TCP는 전송계층(4), IP는 네트워크계층(3)
