# 포트와 다중화

> ⭐ Multiplexing = 여러 앱 → 하나의 전송계층 / Demultiplexing = Port 번호로 올바른 앱에 전달

---

## Port란?

컴퓨터 안에서 실행 중인 프로그램(Process)을 식별하기 위한 번호.

```
IP 주소 → 컴퓨터 찾기
Port 번호 → 프로그램 찾기
```

**실생활 비유:**
- IP 주소 = 아파트 주소 (서울특별시 ○○구 ○○로)
- Port 번호 = 동/호수 (101동 1203호)

---

## 다중화 (Multiplexing)

여러 프로그램이 동시에 데이터를 보내는 과정.

```
Chrome ──┐
Discord ─┤
Steam ───┼──▶ TCP/UDP ──▶ IP ──▶ 네트워크
Zoom ────┘
```

---

## 역다중화 (Demultiplexing)

수신 측에서 Port 번호를 확인하여 올바른 프로그램으로 전달.

```
네트워크 ──▶ IP ──▶ TCP/UDP ──▶ Port 확인 ──▶ Chrome
                                              Discord
                                              Steam
                                              Zoom
```

---

## 포트를 이용한 다중화/역다중화 예시

| 서비스 | 프로토콜 | 포트 |
|--------|----------|------|
| SMTP | TCP | 25 |
| HTTP | TCP | 80 |
| DNS | UDP | 53 |
| DHCP | UDP | 67 |

클라이언트가 서버로 요청할 때:
- Source Port: 임시 포트 (예: 3022)
- Destination Port: Well-Known Port (예: 80)

서버가 클라이언트로 응답할 때:
- Source Port: 80
- Destination Port: 3022 (요청 시의 Source Port)

---

## 클라이언트/서버 메시지 교환 예시

**요청 (클라이언트 → 서버):**
```
출발지: 177.41.72.6  포트: 3022
목적지: 41.199.222.3 포트: 80
```

**응답 (서버 → 클라이언트):**
```
출발지: 41.199.222.3 포트: 80
목적지: 177.41.72.6  포트: 3022
```

요청과 응답에서 출발지/목적지가 서로 바뀐다.

---

## Socket Address

IP 주소와 Port 번호를 함께 사용하는 것.

```
192.168.0.10:80
203.0.113.10:443
```

---

## 자주 틀리는 부분

❌ IP 주소가 프로그램을 구분한다 → Port 번호가 구분  
❌ 서버와 클라이언트는 항상 같은 Port를 사용한다 → 서버는 Well-Known Port, 클라이언트는 Dynamic Port 사용
