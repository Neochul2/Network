# UDP (User Datagram Protocol)

> ⭐ UDP Header = 8Byte / Checksum 선택적 / Virtual Header는 실제 전송 안 됨

---

## UDP란?

전송계층의 비연결형(Connectionless) 프로토콜. 빠른 데이터 전달이 목적.

---

## UDP가 하는 일

1. 상위 계층 데이터 전송
2. UDP 메시지 캡슐화
3. IP로 전달

---

## UDP가 하지 않는 일

1. 연결 수립 (연결 없이 바로 전송)
2. 승인(ACK) 전송
3. 손실 데이터 재전송
4. 순서 보장
5. 흐름 제어
6. 혼잡 제어

---

## UDP 메시지 포맷 (헤더 8Byte)

```
0               16              32
+---------------+---------------+
|   출발지 포트  |   목적지 포트  |
+---------------+---------------+
|      길이      |    체크섬     |
+---------------+---------------+
|              데이터            |
+-------------------------------+
```

| 필드 | 크기 | 설명 |
|------|------|------|
| 출발지 포트 | 2Byte | 송신 프로그램 포트 |
| 목적지 포트 | 2Byte | 수신 프로그램 포트 |
| 길이 | 2Byte | UDP 헤더 + 데이터 전체 길이 |
| 체크섬 | 2Byte | 오류 검출 (IPv4에서는 선택적, IPv6에서는 필수) |

---

## UDP 가상헤더 (Pseudo Header)

체크섬 계산 시에만 사용하는 임시 헤더. **실제로 전송되지 않는다.**

```
+-------------------------------+
| 출발지 주소 (IP 헤더에서)      |
+-------------------------------+
| 목적지 주소 (IP 헤더에서)      |
+-------------------------------+
| 예약됨 | 프로토콜 | UDP 길이   |
+-------------------------------+
```

- 가상헤더는 계산에만 쓰이고 버려짐
- 실제 전송되는 것은 UDP 헤더 + 데이터뿐

---

## UDP 대표 서비스

DNS, DHCP, TFTP, SNMP, NFS(초기), VoIP, 온라인 게임, 실시간 스트리밍, BOOTP, RIP

---

## 자주 틀리는 부분

❌ UDP는 오류 발생 시 자동 재전송한다 → 재전송은 TCP의 기능  
❌ Virtual Header도 네트워크로 전송된다 → Checksum 계산에만 사용됨  
❌ UDP Checksum은 항상 필수다 → IPv4에서는 선택적
