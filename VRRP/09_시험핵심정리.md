|---------|---------|
| Packet 전달 | Advertisement 수신 |

# 시험에 자주 나오는 내용

✔ VRRP는 Gateway 이중화 프로토콜이다.

✔ OSI Layer 3(Network Layer)에서 동작한다.

✔ Priority가 가장 높은 Router가 Master가 된다.

✔ Priority 255는 Virtual IP Owner이다.

✔ Master만 Advertisement를 전송한다.

✔ Backup은 Advertisement를 감시한다.

✔ 목적지 Multicast는 224.0.0.18이다.

✔ IP Protocol Number는 112이다.

✔ TTL은 255이다.

✔ Advertisement 기본 주기는 1초이다.

✔ Backup은 Advertisement가 일정 시간 도착하지 않으면 Master Down으로 판단한다.

✔ Backup Router가 Master로 승격된다(Failover).

✔ Virtual IP와 Virtual MAC은 변경되지 않는다.

# 최종 핵심

VRRP는 Gateway 장애에 대비하기 위한 **표준 Gateway 이중화 프로토콜**이다.

동일한 VRID를 가진 Router들이 하나의 Virtual Router를 구성하고, Priority가 가장 높은 Router가 Master가 된다.

Master는 Advertisement Packet을 주기적으로 전송하며, Backup은 이를 감시하다가 Advertisement를 일정 시간 동안 수신하지 못하면 Master 장애로 판단하고 자동으로 Master로 승격(Failover)한다.

이 과정에서도 Virtual IP와 Virtual MAC은 유지되므로 사용자는 Gateway 변경 없이 계속 통신할 수 있다.
