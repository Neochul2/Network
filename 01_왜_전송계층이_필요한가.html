TCP/IP 네트워크

PART 1. 왜 전송계층(Transport Layer)이 필요한가?

학습 목표

  • 전송계층이 필요한 이유를 이해한다.
  • IP 계층만으로는 해결할 수 없는 문제를 설명할 수 있다.
  • 전송계층의 주요 역할을 이해한다.
  • TCP와 UDP가 등장한 배경을 이해한다.

왜 전송계층을 먼저 배우는가?

TCP/IP를 처음 공부하는 학생들은 대부분 TCP와 UDP의 특징부터 외우기 시작합니다.

하지만 그렇게 공부하면

  • 왜 TCP가 필요한지
  • 왜 UDP가 존재하는지
  • 왜 ACK가 필요한지
  • 왜 Sliding Window가 등장하는지

를 이해하기 어렵습니다.

모든 기술은 어떤 문제를 해결하기 위해 만들어졌습니다.

전송계층도 마찬가지입니다.

따라서 이 장에서는 TCP나 UDP를 배우기 전에,

왜 전송계층이 필요한가?

부터 이해합니다.

인터넷에서 데이터는 어떻게 전달될까?

우리가 인터넷을 사용할 때는 다양한 프로그램이 동시에 동작합니다.

예를 들어 하나의 컴퓨터에서는

Chrome

Discord

Steam

Zoom

Outlook

모두 동시에 인터넷을 사용할 수 있습니다.

사용자는 단순히 프로그램을 실행하지만,

운영체제는 각각의 프로그램이 보내는 데이터를 네트워크로 전달해야 합니다.

데이터는 계층을 따라 이동한다.

그렇다면

IP 계층은 어떤 역할을 할까요?

IP 계층의 역할

IP(Internet Protocol)는

목적지 컴퓨터까지 데이터를 전달하는 역할을 합니다.

예를 들어

목적지

192.168.0.10

이라는 IP 주소가 있다면,

IP는 해당 컴퓨터를 찾아 데이터를 전달합니다.

즉,

IP는

컴퓨터(Host)를 찾는 역할을 수행합니다.

그런데 문제가 있다.

IP는 컴퓨터까지는 찾아갈 수 있습니다.

하지만

컴퓨터 안에는 여러 프로그램이 동시에 실행되고 있습니다.

예를 들어

192.168.0.10

├── Chrome
├── Discord
├── Steam
├── Zoom
└── Outlook

IP는

이 중

어느 프로그램이 데이터를 받아야 하는지

알지 못합니다.

첫 번째 문제

프로그램을 구분할 수 없다.

예를 들어

웹 브라우저가 요청한 데이터를

게임 프로그램이 받아버린다면

정상적인 통신이 이루어질 수 없습니다.

즉,

컴퓨터까지 찾는 것만으로는

통신이 완성되지 않습니다.

프로그램까지 찾아야 합니다.

이러한 문제를 해결하는 계층

이러한 문제를 해결하기 위해

전송계층(Transport Layer)이 존재합니다.

전송계층은

IP가 하지 못하는 기능을 제공합니다.

대표적으로

  • 프로그램 구분
  • 신뢰성 제공
  • 데이터 순서 유지
  • 흐름 제어
  • 연결 관리

를 수행합니다.

전송계층의 주요 역할

1. 프로그램 식별

Port 번호를 이용하여

데이터를 올바른 프로그램으로 전달합니다.

예)

  • HTTP → Port 80
  • HTTPS → Port 443
  • DNS → Port 53

2. 신뢰성 제공

데이터가 손실되면

다시 전송합니다.

3. 데이터 순서 유지

순서가 바뀌면

원래 순서대로 재조립합니다.

4. 흐름 제어

수신자가 처리할 수 있는 속도에 맞추어

데이터를 전송합니다.

5. 연결 관리

필요한 경우

통신 전에 연결을 생성하고

통신이 끝나면 연결을 종료합니다.

전송계층은

응용프로그램과

IP 계층 사이에서

통신을 관리하는 역할을 수행합니다.

전송계층의 대표 프로토콜

전송계층에는 대표적으로

두 개의 프로토콜이 있습니다.

  • TCP
  • UDP

하지만

두 프로토콜은

목적이 서로 다릅니다.

TCP는

정확성을 우선합니다.

UDP는

속도를 우선합니다.

왜 TCP와 UDP 두 개가 존재할까?

모든 인터넷 서비스가

같은 특성을 요구하지는 않습니다.

예를 들어

파일 다운로드는

데이터가 하나도 틀리면 안 됩니다.

반면

실시간 음성 통화는

조금의 데이터 손실보다

지연 시간이 더 큰 문제입니다.

따라서

전송계층은

두 가지 다른 방식의 프로토콜을 제공합니다.

  • TCP : 신뢰성 중심
  • UDP : 속도 중심

다음 장에서는

두 프로토콜의 차이를 자세히 살펴보겠습니다.

시험 포인트

  • 전송계층은 OSI 7계층의 4계층이다.
  • IP는 Host를 식별한다.
  • Port는 Process를 식별한다.
  • 전송계층은 프로그램 간 통신을 담당한다.
  • 대표적인 전송계층 프로토콜은 TCP와 UDP이다.

자주 틀리는 부분

❌ IP가 프로그램을 구분한다.

→ 아니다.

프로그램을 구분하는 것은 전송계층의 Port 번호이다.

❌ TCP와 IP는 같은 계층이다.

→ 아니다.

TCP는 전송계층,

IP는 네트워크 계층이다.

핵심 요약

  • IP는 컴퓨터까지 데이터를 전달한다.
  • 전송계층은 프로그램까지 데이터를 전달한다.
  • 전송계층은 신뢰성, 순서 유지, 흐름 제어, 연결 관리 등을 담당한다.
  • 대표적인 전송계층 프로토콜은 TCP와 UDP이다.
