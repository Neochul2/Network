# 왜 TCP와 UDP를 배우는가?

앞 장에서는

전송계층이 필요한 이유를 배웠습니다.

전송계층은

IP가 해결하지 못하는 문제를 해결하기 위해 존재합니다.

하지만

전송계층에는

두 개의 대표 프로토콜이 있습니다.

- TCP
- UDP

왜 하나가 아니라

두 개가 존재할까요?

---

# 모든 서비스가 같은 요구사항을 가지는 것은 아니다.

인터넷에는 다양한 서비스가 존재합니다.

예를 들어

- 웹 브라우징
- 파일 다운로드
- 이메일
- 인터넷 뱅킹
- 온라인 게임
- 실시간 음성 통화
- 실시간 영상 스트리밍

모두 데이터를 전송하지만

요구사항은 서로 다릅니다.

---

# 정확성이 중요한 서비스

다음과 같은 서비스는

데이터가 하나라도 틀리면 안 됩니다.

- 인터넷 뱅킹
- 파일 다운로드
- 이메일
- 쇼핑몰 결제
- 원격 접속

예를 들어

100MB 파일을 다운로드했는데

중간 데이터가 손실되었다면

파일은 사용할 수 없습니다.

이러한 서비스는

속도가 조금 느려도

정확하게 전달되어야 합니다.

---

# 속도가 중요한 서비스

반대로

다음과 같은 서비스는

실시간성이 중요합니다.

- 온라인 게임
- VoIP
- 화상회의
- IPTV
- 실시간 방송

예를 들어

영상 통화 중

패킷 하나가 손실되는 것은

크게 문제가 되지 않습니다.

하지만

3초 늦게 도착하면

대화 자체가 불가능해집니다.

즉

조금의 손실보다

빠른 전달이 중요합니다.

---

# 하나의 프로토콜로는 해결할 수 없다.

모든 통신을

정확성 중심으로 처리하면

실시간 서비스가 느려집니다.

반대로

모든 통신을

속도 중심으로 처리하면

파일 다운로드가 손상됩니다.

따라서

전송계층은

두 가지 프로토콜을 제공합니다.

```text
정확성

↓

TCP

---------------------

속도

↓

UDP
```

---

# TCP란?

TCP

(Transmission Control Protocol)

는

신뢰성 있는 데이터 전송을 위한

연결형(Connection-Oriented)

프로토콜입니다.

TCP는

다음 기능을 제공합니다.

- 연결 생성
- 연결 종료
- 순서 보장
- 재전송
- ACK
- 흐름 제어
- 혼잡 제어

즉

데이터를

정확하게 전달하는 것이 목적입니다.

---

# UDP란?

UDP

(User Datagram Protocol)

는

빠른 데이터 전송을 위한

비연결형(Connectionless)

프로토콜입니다.

UDP는

다음 특징을 가집니다.

- 연결 생성 없음
- ACK 없음
- 재전송 없음
- 순서 보장 없음
- 흐름 제어 없음
- 혼잡 제어 없음

대신

매우 빠르게 데이터를 전달합니다.

---

# TCP와 UDP 비교

|항목|TCP|UDP|
|---|---|---|
|연결|연결형|비연결형|
|신뢰성|제공|제공하지 않음|
|재전송|지원|지원하지 않음|
|ACK|사용|사용하지 않음|
|순서 보장|보장|보장하지 않음|
|흐름 제어|지원|지원하지 않음|
|혼잡 제어|지원|지원하지 않음|
|속도|상대적으로 느림|빠름|

# TCP를 사용하는 서비스

대표적인 TCP 기반 서비스

# UDP를 사용하는 서비스

대표적인 UDP 기반 서비스

- DNS
- DHCP
- VoIP
- IPTV
- 온라인 게임
- 실시간 스트리밍
- NTP
- SNMP

이 서비스들은

빠른 응답이 중요합니다.

# TCP와 UDP의 공통점

TCP와 UDP는

모두 전송계층 프로토콜입니다.

또한

둘 다

Port 번호를 사용하여

응용프로그램을 구분합니다.

즉

차이는

신뢰성 제공 여부입니다.

TCP와 UDP는

모두 데이터를

그렇다면

같은 컴퓨터에서

Chrome

이 동시에 실행된다면

어떻게 올바른 프로그램으로

데이터를 전달할까요?

이 문제를 해결하는 것이

Port 번호입니다.

다음 장에서는

Port와

Multiplexing,

Demultiplexing의 원리를 학습합니다.

# 시험 포인트

- TCP는 연결형 프로토콜이다.
- UDP는 비연결형 프로토콜이다.
- TCP는 신뢰성을 제공한다.
- UDP는 속도를 우선한다.
- TCP는 ACK를 사용한다.
- UDP는 재전송하지 않는다.

---

# 자주 틀리는 부분

❌ TCP가 항상 더 좋은 프로토콜이다.

→ 아니다.

서비스의 목적에 따라 TCP와 UDP를 선택한다.

---

❌ UDP는 오류가 발생하면 자동으로 다시 보낸다.

→ 아니다.

UDP는 재전송 기능을 제공하지 않는다.

---

# 핵심 요약

- TCP는 정확한 데이터 전달을 위한 프로토콜이다.
- UDP는 빠른 데이터 전달을 위한 프로토콜이다.
- 인터넷 서비스의 특성에 따라 TCP와 UDP를 선택한다.
- 두 프로토콜은 경쟁 관계가 아니라 서로를 보완하는 관계이다.
