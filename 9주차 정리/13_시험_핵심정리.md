# 전송계층 핵심 개념

전송계층(Transport Layer)은

응용프로그램(Application)과

네트워크 계층(IP) 사이에서

프로그램 간 통신을 담당하는 계층입니다.

주요 역할은 다음과 같습니다.

- 프로그램 식별(Port)
- 데이터 전송
- 신뢰성 제공
- 연결 관리
- 흐름 제어
- 오류 복구

대표 프로토콜은

- TCP
- UDP

입니다.

---

# TCP와 UDP 비교

|항목|TCP|UDP|
|---|---|---|
|연결 방식|연결형(Connection-Oriented)|비연결형(Connectionless)|
|신뢰성|제공|제공하지 않음|
|ACK|사용|사용하지 않음|
|재전송|지원|지원하지 않음|
|순서 보장|보장|보장하지 않음|
|흐름 제어|지원|지원하지 않음|
|혼잡 제어|지원|지원하지 않음|
|속도|상대적으로 느림|빠름|
|Header 크기|20~60Byte|8Byte|

---

# Port 번호 정리

Port는

컴퓨터 안에서

프로그램(Process)을 식별하는 번호입니다.

|구분|범위|
|---|---|
|Well-Known Port|0 ~ 1023|
|Registered Port|1024 ~ 49151|
|Dynamic(Private) Port|49152 ~ 65535|

대표적인 Port 번호

|Port|서비스|
|---:|---|
|20, 21|FTP|
|22|SSH|
|23|Telnet|
|25|SMTP|
|53|DNS|
|67|DHCP Server|
|68|DHCP Client|
|80|HTTP|
|110|POP3|
|143|IMAP|
|443|HTTPS|

---

# UDP 핵심 정리

UDP는

- 비연결형
- 빠른 전송
- ACK 없음
- 재전송 없음
- 순서 보장 없음
- Header 크기 8Byte

특징을 가집니다.

대표 서비스

- DNS
- DHCP
- VoIP
- IPTV
- 온라인 게임
- 실시간 스트리밍

---

# TCP 핵심 정리

TCP는

- 연결형
- 신뢰성 제공
- ACK 사용
- 재전송
- 순서 보장
- 흐름 제어
- 혼잡 제어

기능을 제공합니다.

대표 서비스

- HTTP
- HTTPS
- FTP
- SSH
- SMTP
- POP3
- IMAP

---

# TCP Header 핵심 필드

|필드|역할|
|---|---|
|Source Port|송신 프로그램|
|Destination Port|수신 프로그램|
|Sequence Number|현재 Segment의 첫 번째 Byte 번호|
|ACK Number|다음에 받을 Byte 번호|
|Header Length|Header 크기|
|Control Bits|연결 및 제어|
|Window Size|Flow Control|
|Checksum|오류 검출|
|Urgent Pointer|긴급 데이터|
|Options|추가 기능|

---

# TCP Control Bit

|Bit|역할|
|---|---|
|SYN|연결 생성|
|ACK|수신 확인|
|FIN|정상 종료|
|RST|강제 종료|
|PSH|즉시 전달|
|URG|긴급 데이터|

---

# Sequence Number와 ACK

Sequence Number

→ 현재 Segment의 첫 번째 Byte 번호

ACK Number

→ 다음에 받아야 할 Byte 번호

예)

```text
1 ~ 100 Byte 수신

↓

ACK = 101
```

TCP는

다음 기능을 이용하여

신뢰성을 제공합니다.

```text
Sequence Number

↓

ACK

↓

Timeout

↓

Retransmission

↓

PAR
```

---

# Sliding Window

Sliding Window는

ACK를 기다리지 않고

여러 데이터를

연속으로 전송하는 기법입니다.

```text
전송 가능

[1][2][3][4]

↓

ACK

↓

Window 이동

[5][6][7][8]
```

Flow Control을 수행하기 위해

Window Size를 사용합니다.

|구분|Flow Control|Congestion Control|
|---|---|---|
|대상|수신자|네트워크|
|목적|수신자 보호|네트워크 보호|

---

# 3-Way Handshake

```text
Client                    Server

SYN ---------------------->

      <------------------- SYN + ACK

ACK ---------------------->
```

목적

→ 연결 생성

---

# 4-Way Handshake

```text
Client                    Server

FIN ---------------------->

      <------------------- ACK

      <------------------- FIN

ACK ---------------------->
```

목적

→ 연결 종료

# 최종 핵심 요약

- 전송계층은 프로그램 간 통신을 담당한다.
- TCP는 신뢰성을 제공하는 연결형 프로토콜이다.
- UDP는 빠른 전송을 위한 비연결형 프로토콜이다.
- Port 번호는 프로그램을 식별한다.
- Sequence Number는 Byte 단위 번호이다.
- ACK Number는 다음에 받을 Byte 번호이다.
- Sliding Window는 전송 효율을 높이는 기법이다.
- Flow Control은 수신자를 보호한다.
- Congestion Control은 네트워크를 보호한다.
- TCP는 3-Way Handshake로 연결하고 4-Way Handshake로 종료한다.
- TCP는 FSM을 이용하여 연결 상태를 관리한다.
