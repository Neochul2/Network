# 왜 학습 로드맵이 필요한가?

많은 학생들이 TCP/IP를 공부하면서

- TCP 특징
- UDP 특징
- Header
- ACK
- Sliding Window

를 각각 따로 외웁니다.

하지만 이렇게 공부하면

TCP가 왜 만들어졌는지

왜 ACK가 필요한지

왜 Sliding Window가 등장했는지

이해하기 어렵습니다.

이 교재는

TCP/IP를

**하나의 이야기**

처럼 설명합니다.

---

# 이 교재의 학습 방식

TCP는

수많은 기술이

갑자기 만들어진 것이 아닙니다.

하나의 문제가 생기고

그 문제를 해결하기 위해

다음 기술이 등장했습니다.

즉

```
문제 발생

↓

기술 등장

↓

새로운 문제

↓

새로운 기술 등장
```

이라는 흐름으로 발전했습니다.
