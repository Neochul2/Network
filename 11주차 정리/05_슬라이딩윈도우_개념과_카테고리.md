# Chapter 05 — Sliding Window 개념과 카테고리

---

## 1. Stop-and-Wait의 문제점

TCP가 데이터를 하나 보내고 ACK를 기다리는 방식을 **Stop-and-Wait** 라고 한다.

```
송신자          수신자
  │── Data1 ──▶│
  │◀── ACK ───│
  │── Data2 ──▶│
  │◀── ACK ───│
```

RTT(왕복 지연)가 길수록 대기 시간이 길어져 **성능이 크게 떨어진다.**

---

## 2. Sliding Window란?

ACK를 기다리지 않고 **Window 크기만큼 여러 데이터를 연속으로 전송**하는 기술이다.

```
송신자              수신자
  │── Data1 ──▶│
  │── Data2 ──▶│
  │── Data3 ──▶│
  │◀─── ACK ──│
```

Window가 클수록 한 번에 더 많은 데이터를 전송할 수 있어 **네트워크 효율이 높아진다.**

---

## 3. Window란?

> ACK를 받기 전에 송신자가 연속해서 보낼 수 있는 **최대 데이터 크기(Byte)**

이 값은 수신자가 **광고(Advertise)** 하는 Receive Buffer 여유 공간을 기반으로 결정된다.

```
Receive Buffer : 1000 Byte
Advertised Window : 1000 Byte
→ 송신자는 최대 1000 Byte까지 ACK 없이 전송 가능
```

---

## 4. 송신 카테고리 4가지

TCP는 송신 데이터를 다음 4가지 영역으로 구분하여 관리한다.

```
|──── 1 ────|──── 2 ────|──── 3 ────|──── 4 ────|
  ACK 완료    ACK 대기    전송 가능    전송 불가
```

| 카테고리 | 상태 | 설명 |
|:---:|:---:|:---|
| 1 | ACK 완료 | 전송했고 ACK까지 받은 데이터 |
| 2 | ACK 대기 | 전송했지만 아직 ACK를 받지 못한 데이터 |
| 3 | 전송 가능 | 수신자가 준비됐지만 아직 전송하지 않은 데이터 |
| 4 | 전송 불가 | 수신자가 준비되지 않아 전송할 수 없는 데이터 |

> **Window = 카테고리 2 + 카테고리 3**

---

## 5. 수신 카테고리 3가지

수신 측도 비슷하게 상태를 관리한다.

```
|──── 1+2 ────|──── 3 ────|──── 4 ────|
  수신 + ACK    수신 가능    수신 불가
```

| 카테고리 | 설명 |
|:---:|:---|
| 1+2 | 수신했고 ACK를 전송한 데이터 |
| 3 | 수신하지 않았지만 송신자가 보내도 되는 데이터 |
| 4 | 수신자가 준비되지 않아 받을 수 없는 데이터 |

---

## 6. Window가 이동하는 원리

ACK가 도착하면 카테고리 2의 데이터가 카테고리 1로 이동하면서 **Window가 앞으로 밀린다.**

```
ACK 전

[──2──|──3──|──4──]
  ^Window^

ACK 수신

      [──2──|──3──|──4──]
        ^Window 이동^
```

Window가 앞으로 **미끄러지듯(Slide)** 이동하기 때문에 **Sliding Window** 라고 한다.

---

## 7. ACK와 Sliding Window의 관계

ACK는 단순히 "잘 받았습니다" 가 아니다.

동시에 **"다음 데이터를 보내도 됩니다"** 라는 의미도 담고 있다.

```
ACK 도착
    │
    ▼
Window 이동
    │
    ▼
다음 데이터 전송 가능
```

---

## 8. Window와 Buffer의 관계

```
Receive Buffer 여유 감소 → Window 감소 → 송신 속도 감소
Receive Buffer 여유 증가 → Window 증가 → 송신 속도 증가
```

Buffer가 가득 차면 Window = 0 이 되어 송신자는 전송을 멈춘다. → **Zero Window**

---

## 시험 포인트

- Sliding Window = ACK 없이 Window 크기만큼 연속 전송
- Window = 카테고리 2 + 카테고리 3
- ACK 도착 → Window 이동
- Window = 수신자 Receive Buffer 여유 공간
- Sliding Window는 TCP Flow Control의 핵심 기술

---

## 핵심 요약

| 항목 | 내용 |
|:---|:---|
| Sliding Window | ACK 없이 여러 데이터를 연속 전송하는 기술 |
| Window 크기 | 수신자 Receive Buffer 여유 공간 기반 |
| 카테고리 2+3 | 현재 송신 Window 영역 |
| ACK 역할 | 수신 확인 + Window 이동 신호 |
| Zero Window | Buffer 가득 참 → Window = 0 → 전송 중지 |
