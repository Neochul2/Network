# 2. TCP Header 핵심 필드

★★★★★

반드시 암기

| 필드 | 역할 |
|------|------|
| Source Port | 송신 프로그램 |
| Destination Port | 수신 프로그램 |
| Sequence Number | 데이터 순서 |
| ACK Number | 다음 받을 Byte |
| Window | 수신 가능한 크기 |
| Checksum | 오류 검출 |

Control Bits 6가지

| 비트 | 의미 |
|------|------|
| URG | 긴급 데이터 |
| ACK | 승인 번호 유효 |
| PSH | 즉시 전달 |
| RST | 연결 초기화 |
| SYN | 연결 수립 |
| FIN | 연결 종료 |

# 3. Pseudo Header

★★★★★

실제 전송되지 않는다.

Checksum 계산에만 사용한다.

계산 후 즉시 제거된다.

포함 정보

- Source IP

- Destination IP

- Protocol (TCP = 6)

- TCP Length

```
Pseudo Header

+

TCP Header

+

Payload

↓

Checksum 계산

↓

Pseudo Header 제거

↓

TCP Segment 전송
```

---

# 4. MSS

★★★★★

MSS

=

Data만

Header 제외

공식

```
MSS

=

MTU

-

IP Header

-

TCP Header
```

Ethernet

```
1460Byte
```

기본 MSS

```
536Byte

(IPv4 최소 MTU 576 - IP 20 - TCP 20)
```

MSS가 너무 작으면 → Header 오버헤드 증가

MSS가 너무 크면 → Fragmentation 발생

# 5. Sliding Window

★★★★★

ACK를 기다리지 않고

여러 데이터를

전송하는 기술

Window

=

Receiver Buffer

ACK가 오면

Window 이동

송신 카테고리

| 카테고리 | 의미 |
|---------|------|
| 1 | 전송 + ACK 완료 |
| 2 | 전송 + ACK 대기 |
| 3 | 전송 가능 |
| 4 | 전송 불가 |

Window = 카테고리 2 + 3

# 8. Retransmission

★★★★★

전송

↓

재전송 큐 저장

↓

ACK

↓

삭제

ACK 없음

↓

↓

---

RTT와 RTO

```
RTT

=

세그먼트 전송 ~ ACK 수신까지 시간

↓

```

RTT를 측정하여 RTO를 동적으로 계산한다.

---

Fast Retransmit

```
동일한 ACK 3회 수신

↓

Timeout 없이 즉시 재전송
```

Timeout보다 빠르게 복구한다.

---

재전송 정책

| 정책 | 장점 | 단점 |
|------|------|------|
| Timeout 세그먼트만 재전송 | 대역폭 절약 | 복구 느림 |
| 모든 미승인 세그먼트 재전송 | 복구 빠름 | 불필요한 재전송 |

현대 TCP는 SACK + Fast Retransmit을 함께 사용한다.

# 11. SWS

★★★★★

작은 데이터를

계속 보내는 현상

문제

- 성능 저하

- Header 오버헤드 증가

- 대역폭 낭비

해결

| 방법 | 적용 위치 | 역할 |
|------|---------|------|
| Clark Solution | 수신 측 | 작은 Window 광고 방지 |
| Nagle Algorithm | 송신 측 | 작은 데이터를 모아서 전송 |

```
Name Space

↓
```

---

Name Registration 3가지 방법

| 방법 | 설명 |
|------|------|
| Table 등록 | 관리자가 테이블 직접 수정 |
| Broadcast 등록 | 네트워크 전체에 이름 충돌 확인 |

DNS는 Database 등록 방식을 사용한다.

- Zone = DNS 서버가 실제 관리하는 영역

# 빈출 OX 문제

### O

TCP Segment는 Header와 Data로 구성된다.

---

### O

Pseudo Header는 실제 전송되지 않는다.

---

### X

MSS에는 TCP Header가 포함된다.

→ 포함되지 않는다. Data(Payload)만이다.

---

### O

ACK 번호는 다음에 받을 Byte 번호이다.

---

### O

Sliding Window는 ACK를 기다리지 않고 여러 데이터를 전송하기 위한 기술이다.

---

### O

Window Size는 Receiver Buffer의 남은 공간을 의미한다.

---

### O

Window가 0이면 Zero Window이다.

---

### O

SACK는 손실된 Segment만 재전송하도록 도와준다.

---

### O

Fast Retransmit은 동일한 ACK를 3회 수신하면 즉시 재전송한다.

---

### O

DNS는 Domain Name을 IP Address로 변환한다.

---

### O

DNS Name Registration 방법에는 Table, Broadcast, Database가 있다.

---

### O

Domain과 Zone은 서로 다른 개념이다.
