# 1. TCP Header란?

TCP Header는 TCP가 데이터를 신뢰성 있게 전송하기 위해 사용하는 **제어
정보(Control Information)의 집합**이다.

TCP Segment는 다음과 같이 구성된다.

``` text
TCP Header

+

Payload(Data)
```

TCP Header에는 다음 기능을 수행하기 위한 정보가 저장된다.

-   연결 관리(Connection Management)
-   순서 관리(Sequence Control)
-   오류 검출(Error Detection)
-   흐름 제어(Flow Control)
-   재전송(Retransmission)

즉, TCP Header는 데이터를 안전하고 정확하게 전달하기 위한 핵심 정보가
저장되는 영역이다.

------------------------------------------------------------------------

# 2. TCP Header 구조

TCP Header는 최소 **20Byte**, 최대 **60Byte**까지 사용할 수 있다.

``` text
  0                   15 16                  31
 +----------------------+----------------------+
 | Source Port          | Destination Port     |
 +----------------------+----------------------+
 |               Sequence Number               |
 +---------------------------------------------+
 |          Acknowledgement Number             |
 +----+----------+-----------+-----------------+
 |DOFF| Reserved |   Flags   |     Window      |
 +----------------------+----------------------+
 |     Checksum         |   Urgent Pointer     |
 +----------------------+----------------------+
 |     Options (0~40Byte) + Padding            |
 +---------------------------------------------+
 |              Payload(Data)                  |
 +---------------------------------------------+
```

각 필드는 TCP의 신뢰성 있는 통신을 위해 서로 협력하여 동작한다.

# 3. Source Port

## 정의

세그먼트를 송신하는 프로그램(Process)의 포트 번호이다.

-   크기 : **16bit**
-   일반적으로 클라이언트는 **Ephemeral Port**를 사용한다.

### 예시

``` text
Chrome
   │
   ▼
Source Port : 52341
   │
   ▼
Destination Port : 443
   │
   ▼
HTTPS Server
```

### 왜 필요한가?

응답을 어떤 프로그램이 받아야 하는지 구분하기 위해 사용한다.

# 6. Sequence Number

TCP 세그먼트에 포함된 **첫 번째 Byte의 순서 번호**이다.

``` text
Hello

↓

H  e  l  l  o

100 101 102 103 104

Sequence Number = 100
```

역할

-   데이터 순서 보장
-   재전송
-   중복 제거

> 교수님 강조 ★★★★★

------------------------------------------------------------------------

# 7. Acknowledgement Number

ACK Number는 **수신자가 다음에 받고 싶은 Byte 번호**를 의미한다.

``` text
1~100 Byte 수신 완료

↓

ACK = 101
```

의미

> 1\~100 Byte는 정상적으로 수신했으며, 다음에는 101 Byte부터 보내십시오.

TCP는 **누적 승인(Cumulative Acknowledgement)** 방식을 사용한다.

> 교수님 강조 ★★★★★

------------------------------------------------------------------------

# 9. Reserved

미래 기능 확장을 위해 예약된 필드이며 현재는 항상 **0**이다.

------------------------------------------------------------------------

# 10. Control Bits (Flags)

  비트   설명
  ------ -------------------------------------------------
  URG    Urgent Pointer가 유효함을 의미
  ACK    ACK Number가 유효함을 의미
  PSH    데이터를 가능한 빨리 애플리케이션에 전달
  RST    비정상 연결 종료 또는 존재하지 않는 연결 초기화
  SYN    ISN 교환 및 연결 시작
  FIN    정상적인 연결 종료

------------------------------------------------------------------------

# 11. Window

수신 가능한 버퍼 크기(Byte)를 나타낸다.

TCP **Flow Control**의 핵심 필드이다.

> 교수님 강조 ★★★★★

# 12. Checksum

TCP Header와 Payload의 오류를 검출한다.

Checksum 계산에는 다음이 포함된다.

-   TCP Header
-   Payload(Data)
-   Pseudo Header

------------------------------------------------------------------------

# 13. Urgent Pointer

URG 비트가 1일 때만 사용한다.

긴급 데이터의 끝(Byte 위치)를 나타낸다.

실무에서는 거의 사용되지 않는다.

------------------------------------------------------------------------

# 14. Options

  옵션           설명
  -------------- -----------------------
  MSS            최대 Payload 크기
  Window Scale   Window 확장
  SACK           선택적 승인
  Timestamp      RTT 측정 및 PAWS 지원

------------------------------------------------------------------------

# 15. Padding

Options 길이가 32bit의 배수가 아니면 0을 채워 Header 길이를 맞춘다.

# 핵심 요약

-   TCP Header는 신뢰성 있는 통신을 위한 제어 정보를 저장한다.
-   Source/Destination Port는 프로그램을 식별한다.
-   Sequence Number는 첫 번째 Byte 번호이다.
-   ACK Number는 다음에 받고 싶은 Byte 번호이다.
-   Window는 흐름 제어를 수행한다.
-   Checksum은 Pseudo Header를 포함하여 계산한다.
-   Control Bits는 TCP 연결의 생성, 유지, 종료를 제어한다.
