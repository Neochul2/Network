# TCP 가상헤더(Pseudo Header)

------------------------------------------------------------------------

# 학습 목표

이번 장에서는 TCP Checksum 계산에 사용되는 **TCP 가상헤더(Pseudo
Header)** 를 학습한다.

학습 후에는 다음 내용을 설명할 수 있어야 한다.

-   TCP 가상헤더란 무엇인가?
-   왜 필요한가?
-   실제 전송되는가?
-   Checksum은 어떻게 계산되는가?
-   UDP 가상헤더와의 차이점

------------------------------------------------------------------------

# 1. TCP 가상헤더란?

TCP 가상헤더(Pseudo Header)는 TCP Checksum 계산을 위해 **임시로 생성되는
논리적인 Header**이다.

> **Pseudo Header는 실제 네트워크로 전송되지 않는다.**

Checksum 계산이 끝나면 즉시 제거된다.

------------------------------------------------------------------------

# 2. 왜 필요한가?

TCP Checksum은 TCP Header와 Payload뿐만 아니라 **IP 계층의 일부
정보**까지 함께 검사한다.

이렇게 하면 출발지와 목적지 IP 주소가 잘못되는 오류까지 검출할 수 있다.

즉, TCP는 다음 정보를 임시 Header로 만들어 Checksum 계산에 포함한다.

-   Source IP Address
-   Destination IP Address
-   Protocol
-   TCP Length

------------------------------------------------------------------------

# 3. Pseudo Header 구조

``` text
+-------------------------------+
| Source IP Address             |
+-------------------------------+
| Destination IP Address        |
+-------------------------------+
| Reserved                      |
+-------------------------------+
| Protocol (TCP = 6)            |
+-------------------------------+
| TCP Length                    |
+-------------------------------+
```

------------------------------------------------------------------------

# 4. 각 필드 설명

## Source IP Address

송신자의 IPv4 주소

예)

``` text
192.168.0.10
```

------------------------------------------------------------------------

## Destination IP Address

수신자의 IPv4 주소

예)

``` text
142.250.206.78
```

------------------------------------------------------------------------

## Reserved

항상 **0**으로 설정되는 예약 필드이다.

------------------------------------------------------------------------

## Protocol

상위 계층 프로토콜 번호를 의미한다.

  프로토콜     번호
  ---------- ------
  TCP             6
  UDP            17

------------------------------------------------------------------------

## TCP Length

TCP Header와 Payload(Data)의 전체 길이이다.

> IP Header 길이는 포함하지 않는다.

------------------------------------------------------------------------

# 5. Checksum 계산 과정

``` text
TCP Header
+
Payload(Data)
+
Pseudo Header
        │
        ▼
 Checksum 계산
        │
        ▼
 Header에 Checksum 저장
        │
        ▼
 Pseudo Header 제거
        │
        ▼
 TCP Segment 전송
```

Pseudo Header는 계산이 끝나면 삭제된다.

------------------------------------------------------------------------

# 6. 실제 전송되는가?

아니다.

실제로 네트워크로 전송되는 것은 다음뿐이다.

``` text
TCP Header
+
Payload(Data)
```

Pseudo Header는 메모리에서만 사용되는 임시 구조이다.

------------------------------------------------------------------------

# 7. 교수님 수업 포인트

-   Pseudo Header는 임시 Header이다.
-   Checksum 계산에만 사용한다.
-   계산이 끝나면 제거된다.
-   Wireshark에서는 보이지 않는다.
-   학생들이 자주 헷갈리는 개념이다.

------------------------------------------------------------------------

# 8. UDP와 비교

UDP 역시 Checksum 계산을 위해 Pseudo Header를 사용한다.

TCP와 UDP 모두 Source IP, Destination IP, Protocol, Length를 이용하지만
TCP는 신뢰성 있는 전송을 위해 Checksum의 중요성이 더욱 크다.

------------------------------------------------------------------------

# 9. 왜 IP 주소까지 검사하는가?

예를 들어

``` text
192.168.0.10
        │
        ▼
192.168.0.20
```

으로 보내야 하는데 IP 주소가 손상되면 TCP Header와 Payload만 검사해서는
오류를 발견하지 못할 수 있다.

따라서 Source IP와 Destination IP를 함께 포함하여 Checksum을 계산한다.

------------------------------------------------------------------------

# 10. Wireshark에서는?

Wireshark에서는 Pseudo Header를 볼 수 없다.

이미 Checksum 계산 후 제거되었기 때문이다.

Wireshark에서는 다음만 확인할 수 있다.

-   TCP Header
-   Payload(Data)
-   Checksum

------------------------------------------------------------------------

# 시험 포인트

-   Pseudo Header는 실제 전송되지 않는다.
-   Checksum 계산에만 사용한다.
-   Source IP와 Destination IP를 포함한다.
-   Protocol(TCP=6)을 포함한다.
-   TCP Length를 포함한다.

------------------------------------------------------------------------

# 암기법

> **Pseudo Header = Checksum 계산용 임시 Header**
