# Sliding Window란?

TCP는 데이터를 한 개씩 보내고 ACK를 기다리는 방식이 아니다.

만약

```text
데이터 1개 전송

↓

ACK 대기

↓

데이터 1개 전송

↓

ACK 대기
```

이런 방식이라면

네트워크를 매우 비효율적으로 사용하게 된다.

그래서 TCP는

**ACK를 기다리지 않고 여러 개의 데이터를 연속해서 전송**할 수 있도록 Sliding Window 기법을 사용한다.

---

# 왜 Sliding Window가 필요한가?

예를 들어

1000Byte를 전송한다고 가정하자.

ACK를 받을 때까지 기다리면

```text
100Byte 전송

↓

ACK

↓

100Byte 전송

↓

ACK
```

총 10번을 반복해야 한다.

하지만 Window가 500Byte라면

```text
500Byte 전송

↓

500Byte 전송

↓

ACK
```

처럼 훨씬 효율적으로 데이터를 보낼 수 있다.

---

# Window란?

Window는

> **상대방의 ACK를 받기 전에 미리 보낼 수 있는 데이터의 최대 크기**

를 의미한다.

즉,

Window가 클수록

한 번에 더 많은 데이터를 보낼 수 있다.

---

# 송신 카테고리

교수님 PDF에서는 데이터를 네 가지 영역으로 구분한다.

## 카테고리 1

이미 전송했고

ACK도 받은 데이터

```
완료
```

---

## 카테고리 2

전송했지만

아직 ACK를 받지 못한 데이터

```
전송 완료

ACK 대기
```

---

## 카테고리 3

수신자는 받을 준비가 되어 있지만

아직 보내지 않은 데이터

```
다음 전송 대상
```

---

## 카테고리 4

아직 전송할 수 없는 데이터

수신자의 Window 밖에 있는 데이터이다.

---

# 그림으로 이해하기

```text
|----1----|----2----|----3----|----4----|

ACK 완료

전송 완료

전송 가능

전송 불가
```

Window는

카테고리2와

카테고리3을

포함하는 영역이다.

---

# 수신 카테고리

수신 측도

송신 측과 동일하게

네 가지 상태를 관리한다.

### 수신 1

정상적으로 받은 데이터

### 수신 2

ACK를 보낸 데이터

### 수신 3

다음에 받을 데이터

### 수신 4

아직 받을 수 없는 데이터

---

# Window는 왜 움직이는가?

ACK가 도착하면

이미 받은 데이터는

Window에서 제거된다.

그러면

새로운 공간이 생긴다.

이 공간만큼

Window가 앞으로 이동한다.

이것이

Sliding Window

라는 이름의 이유이다.

---

# Sliding의 의미

Window가

고정되어 있는 것이 아니라

계속 앞으로

미끄러지듯 이동한다.

```text
전송 전

□□□□□□□□

↓

ACK 수신

□□□□□□□□

↓

Window 이동

□□□□□□□□
```

Window 자체가

앞으로 이동하는 것이다.

---

# ACK와 Sliding Window

ACK가 오면

Window도 이동한다.

즉

ACK는

단순히

"잘 받았습니다."

가 아니라

"이제 다음 데이터를 보내세요."

라는 의미도 가진다.

---

# Window와 Buffer

많은 학생들이 헷갈리는 부분이다.

Window는

수신 측 Buffer의 크기를 의미한다.

예를 들어

Buffer가

1000Byte라면

송신자는

1000Byte까지

ACK 없이 보낼 수 있다.

만약 Buffer가

가득 차면

Window는 작아진다.

---

# 교수님 수업 포인트

교수님께서는 다음 내용을 반복해서 설명하셨다.

✔ Window = Buffer 크기

✔ Window는 계속 움직인다.

✔ ACK가 도착해야 Window가 앞으로 이동한다.

✔ Sliding Window는 TCP의 핵심 기술이다.

# 핵심 요약

- Sliding Window는 TCP의 성능을 높이기 위한 핵심 기술이다.
- ACK를 기다리지 않고 여러 데이터를 전송할 수 있다.
- Window는 수신자의 Buffer 크기를 의미한다.
- ACK가 도착하면 Window가 앞으로 이동한다.
- 다음 장에서는 SND.UNA, SND.NXT, SND.WND, RCV.NXT, RCV.WND 포인터를 이용해 Window가 실제로 어떻게 이동하는지 학습한다.
