# TCP/IP Advanced

# Chapter 06

# SND / RCV 포인터

------------------------------------------------------------------------

# 학습 목표

이번 장에서는 TCP가 Sliding Window를 구현하기 위해 사용하는
**SND(Sender)** 와 **RCV(Receiver)** 포인터를 학습한다.

학습 후에는 다음 내용을 설명할 수 있어야 한다.

-   SND.UNA
-   SND.NXT
-   SND.WND
-   RCV.NXT
-   RCV.WND
-   포인터가 Sliding Window를 구현하는 원리

------------------------------------------------------------------------

# 1. 왜 포인터가 필요한가?

TCP는 **Byte Stream 기반 프로토콜**이다.

데이터를 지속적으로 송수신하면서 다음 정보를 관리해야 한다.

-   어디까지 전송했는가?
-   어디까지 ACK를 받았는가?
-   다음에는 무엇을 전송해야 하는가?
-   상대방은 얼마나 더 받을 수 있는가?

이 정보를 관리하기 위해 TCP는 여러 개의 포인터를 사용한다.

------------------------------------------------------------------------

# 2. 포인터의 종류

``` text
송신(Sender)

SND.UNA
SND.NXT
SND.WND

        │

        ▼

수신(Receiver)

RCV.NXT
RCV.WND
```

------------------------------------------------------------------------

# 3. SND.UNA (Sender Unacknowledged)

**아직 ACK를 받지 못한 가장 오래된 Byte의 Sequence Number**이다.

예)

``` text
1~100 전송
ACK 미도착

SND.UNA = 1
```

ACK=101이 도착하면

``` text
SND.UNA = 101
```

로 이동한다.

------------------------------------------------------------------------

# 쉽게 이해하기

택배를 여러 개 보냈는데 가장 오래 배송 완료 문자를 기다리는 택배가
있다고 생각하면 된다.

가장 오래 기다리는 택배가 바로 **SND.UNA**이다.

------------------------------------------------------------------------

# 4. SND.NXT (Sender Next)

다음에 전송할 Byte의 Sequence Number이다.

예)

``` text
1~100 전송 완료

↓

SND.NXT = 101
```

------------------------------------------------------------------------

# 5. SND.UNA와 SND.NXT의 차이

``` text
SND.UNA
│
├─ 가장 오래된 미승인 Byte

............. (전송했지만 ACK 대기)

SND.NXT
│
└─ 다음에 전송할 Byte
```

두 포인터 사이에는 이미 전송했지만 아직 ACK를 받지 못한 데이터가
존재한다.

------------------------------------------------------------------------

# 6. SND.WND (Sender Window)

송신 가능한 Window의 크기(Byte)이다.

이 값은 상대방이 광고한 Receive Window를 기반으로 결정된다.

예)

``` text
SND.WND = 500Byte
```

이면 ACK 없이 최대 500Byte 범위까지 추가 전송이 가능하다.

------------------------------------------------------------------------

# 7. RCV.NXT (Receiver Next)

수신자가 다음에 받고 싶은 Byte의 번호이다.

예)

``` text
1~100 수신 완료

↓

RCV.NXT = 101
```

------------------------------------------------------------------------

# 8. ACK와 RCV.NXT

ACK Number는 상대방에게 자신의 **RCV.NXT 값**을 알려주는 것이다.

``` text
ACK = 101

↓

101부터 보내십시오.
```

즉 ACK Number와 RCV.NXT는 같은 의미를 서로 다른 관점에서 표현한 값이다.

------------------------------------------------------------------------

# 9. RCV.WND (Receiver Window)

수신 버퍼에 남아 있는 공간(Byte)을 의미한다.

예)

``` text
Receive Buffer : 1000Byte
사용 중 : 600Byte

↓

RCV.WND = 400Byte
```

송신자는 이 값을 참고하여 전송 가능한 데이터 양을 결정한다.

------------------------------------------------------------------------

# 10. 포인터 관계

``` text
SND.UNA
│
├─ 미승인 데이터 시작

SND.NXT
│
├─ 다음 전송 위치

SND.UNA + SND.WND
│
└─ 송신 Window의 끝
```

송신 가능한 데이터는 Window 범위 안에 있어야 한다.

------------------------------------------------------------------------

# 11. ACK가 오면?

예)

``` text
ACK = 101

SND.UNA
1 → 101
```

ACK가 증가하면 Window도 함께 앞으로 이동한다.

이것이 Sliding Window의 핵심 동작이다.

------------------------------------------------------------------------

# 12. 교수님 수업 포인트

반드시 암기해야 하는 포인터

-   **SND.UNA** : 가장 오래된 미승인 Byte
-   **SND.NXT** : 다음 전송 Byte
-   **SND.WND** : 송신 Window 크기
-   **RCV.NXT** : 다음 수신 Byte
-   **RCV.WND** : 남은 Receive Buffer 크기

------------------------------------------------------------------------

# 13. Wireshark에서는?

Wireshark에는 SND.UNA, SND.NXT 등의 이름이 직접 표시되지는 않는다.

다음 값을 이용하여 포인터의 상태를 분석할 수 있다.

-   Sequence Number
-   ACK Number
-   Window Size

------------------------------------------------------------------------

# 14. 실무 예시

``` text
1~1000Byte 전송

↓

ACK = 501

↓

1~500Byte 정상 수신

↓

Window 이동

↓

501Byte부터 계속 전송
```

이 과정이 연결 종료 시까지 반복된다.

------------------------------------------------------------------------

-   RCV.WND : 남은 Receive Buffer 크기

------------------------------------------------------------------------

# 암기법

``` text
UNA → Unacknowledged

NXT → Next

WND → Window
```

# 핵심 요약

-   TCP는 포인터를 이용하여 Byte Stream을 관리한다.
-   SND.UNA는 가장 오래된 미승인 데이터를 가리킨다.
-   SND.NXT는 다음에 전송할 데이터를 가리킨다.
-   SND.WND는 송신 가능한 범위를 결정한다.
-   RCV.NXT는 다음에 수신할 Byte를 나타낸다.
-   RCV.WND는 수신 가능한 Buffer 크기를 나타낸다.
