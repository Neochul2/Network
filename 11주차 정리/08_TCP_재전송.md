# 1. TCP는 왜 재전송이 필요한가?

인터넷에서는 모든 데이터가 항상 정상적으로 도착하지 않는다.

대표적인 원인

-   Packet Loss
-   네트워크 혼잡(Congestion)
-   라우터 장애
-   전송 오류

TCP는 이러한 상황에서도 신뢰성 있는 전송을 위해 재전송(Retransmission)을
수행한다.

------------------------------------------------------------------------

# 2. 재전송의 기본 원리

``` text
데이터 전송
    │
    ▼
Retransmission Queue 저장
    │
    ▼
ACK 대기
    │
 ┌──┴──┐
 ▼     ▼
ACK   Timeout
 │       │
 ▼       ▼
삭제   재전송
```

ACK가 도착하면 Queue에서 제거하고, ACK가 오지 않으면 동일한 Segment를
다시 전송한다.

------------------------------------------------------------------------

# 3. Retransmission Queue

재전송 큐는 ACK를 받지 못한 TCP Segment를 임시 저장하는 공간이다.

``` text
Part1
Part2
Part3
Part4
──────────────
ACK 대기
```

ACK가 도착하면 승인된 Segment부터 삭제된다.

예)

``` text
ACK=201

↓

Part1 삭제
Part2 삭제

↓

Part3
Part4
```

------------------------------------------------------------------------

# 4. Retransmission Timer

세그먼트를 전송하면 Timer가 시작된다.

``` text
Segment 전송

↓

Timer 시작

↓

ACK 도착?

YES → Timer 종료

NO → Timeout
```

------------------------------------------------------------------------

# 5. RTT와 RTO

TCP는 RTT(Round Trip Time)를 측정하여 재전송 시간(RTO)을 동적으로
계산한다.

``` text
Segment 전송

↓

RTT 측정

↓

평균 RTT 계산

↓

RTO 결정

↓

Timeout 여부 판단
```

------------------------------------------------------------------------

# 6. Timeout

``` text
0초  Segment 전송
1초  ACK 없음
2초  ACK 없음

↓

RTO 만료

↓

재전송
```

재전송 후 새로운 Timer가 시작된다.

------------------------------------------------------------------------

# 7. 교수님 예제

``` text
Part1
↓

Part2
↓

Part3 (손실)
↓

Part4
```

수신자는 Part3이 없으므로 계속 같은 ACK를 보낸다.

``` text
ACK = Part3 시작 번호
```

송신자는 Timeout 또는 Fast Retransmit 후 Part3만 다시 전송한다.

------------------------------------------------------------------------

# 8. 누적 ACK

``` text
1~100
101~200
201~300
```

정상 수신 시

``` text
ACK = 301
```

의미

> 1\~300Byte를 모두 받았으며 301부터 보내십시오.

------------------------------------------------------------------------

# 9. Fast Retransmit

현대 TCP는 Timeout을 기다리지 않는 경우가 있다.

``` text
동일한 ACK

3회 수신

↓

Fast Retransmit

↓

즉시 재전송
```

Timeout보다 빠르게 복구한다.

------------------------------------------------------------------------

# 10. 재전송 정책

## 정책 ①

Timeout이 발생한 Segment만 재전송

장점

-   대역폭 절약

단점

-   복구 속도 느림

## 정책 ②

모든 미승인 Segment 재전송

장점

-   복구 빠름

단점

-   불필요한 재전송 증가

현대 TCP는 SACK, Fast Retransmit 등을 함께 사용한다.

# 핵심 요약

-   TCP는 ACK를 받을 때까지 Segment를 Retransmission Queue에 저장한다.
-   ACK가 오면 Queue에서 제거한다.
-   ACK가 오지 않으면 RTO 만료 후 재전송한다.
-   RTT를 기반으로 RTO를 계산한다.
-   현대 TCP는 Fast Retransmit과 SACK를 함께 사용하여 성능을 향상시킨다.
