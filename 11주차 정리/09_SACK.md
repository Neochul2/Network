# TCP/IP Advanced

# Chapter 09

# SACK (Selective Acknowledgement)

------------------------------------------------------------------------

# 학습 목표

이번 장에서는 TCP의 **선택적 승인(Selective Acknowledgement, SACK)** 을
학습한다.

학습 후에는 다음 내용을 설명할 수 있어야 한다.

-   SACK란 무엇인가?
-   왜 SACK가 필요한가?
-   누적 ACK와의 차이
-   SACK의 동작 과정
-   SACK의 장점

------------------------------------------------------------------------

# 1. SACK란?

SACK(Selective Acknowledgement)은 TCP 수신자가 **이미 정상적으로 받은
데이터 구간을 송신자에게 알려주는 기능**이다.

기본 TCP는 **누적 ACK(Cumulative ACK)** 만 사용하지만, SACK는 누적 ACK의
한계를 보완한다.

> SACK는 TCP Header의 **Options 필드**를 사용한다.

------------------------------------------------------------------------

# 2. 누적 ACK의 문제점

다음과 같이 5개의 세그먼트를 전송했다고 가정하자.

``` text
1   2   3   4   5
```

3번 세그먼트만 손실되었다.

``` text
1 ✓
2 ✓
3 ✗
4 ✓
5 ✓
```

수신자는 4와 5를 받았지만 3이 없으므로 계속

``` text
ACK = 3
```

만 보낸다.

송신자는 4와 5가 이미 도착했는지 알 수 없다.

------------------------------------------------------------------------

# 3. 결과

기본 누적 ACK만 사용하면 송신자는

``` text
3
4
5
```

를 다시 전송할 수 있다.

이미 받은 4와 5도 재전송되어 대역폭이 낭비된다.

------------------------------------------------------------------------

# 4. SACK의 등장

SACK를 사용하면 수신자는

``` text
ACK = 3

SACK :
4~5 수신 완료
```

와 같이 알려준다.

송신자는 **손실된 3만 재전송**하면 된다.

------------------------------------------------------------------------

# 5. 동작 과정

## Step 1

``` text
송신
1 2 3 4 5
```

## Step 2

``` text
수신
1 ✓
2 ✓
3 ✗
4 ✓
5 ✓
```

## Step 3

``` text
ACK = 3
SACK = 4~5
```

## Step 4

``` text
송신자는 3만 재전송
```

------------------------------------------------------------------------

# 6. 누적 ACK와 비교

## 누적 ACK

``` text
3 손실

↓

3 4 5 재전송 가능
```

## SACK

``` text
3 손실

↓

3만 재전송
```

------------------------------------------------------------------------

# 7. 장점

-   불필요한 재전송 감소
-   대역폭 절약
-   처리량(Throughput) 향상
-   대용량 파일 전송 성능 향상

------------------------------------------------------------------------

# 8. SACK는 어디에 저장되는가?

SACK 정보는 **TCP Header의 Options 필드**에 저장된다.

기본 20Byte Header에는 포함되지 않는다.

------------------------------------------------------------------------

# 9. 교수님 수업 포인트

-   기본 TCP는 누적 ACK를 사용한다.
-   SACK는 이미 받은 데이터 구간도 알려준다.
-   손실된 Segment만 재전송하도록 도와준다.

------------------------------------------------------------------------

# 10. Wireshark에서는?

TCP Options에서 다음과 같은 항목을 확인할 수 있다.

``` text
SACK Permitted
```

또는

``` text
SACK Blocks
```

이를 통해 어떤 데이터 구간을 이미 수신했는지 확인할 수 있다.

------------------------------------------------------------------------

# 11. 실무 예시

1GB 파일 다운로드 중 일부 Segment만 손실되었다면,

SACK를 사용하면 손실된 Segment만 재전송하므로 다운로드 성능이 향상된다.

------------------------------------------------------------------------

# 12. 누적 ACK와 SACK 비교

  구분        누적 ACK                   SACK
  ----------- -------------------------- -----------------------
  승인 방식   누적 승인                  선택적 승인
  재전송      여러 Segment 재전송 가능   손실 Segment만 재전송
  대역폭      비효율적                   효율적
  성능        보통                       우수

------------------------------------------------------------------------

# 시험 포인트

-   SACK = Selective Acknowledgement
-   누적 ACK의 단점을 보완
-   이미 받은 데이터 구간을 알림
-   손실된 Segment만 재전송
-   TCP Options 필드 사용

------------------------------------------------------------------------


# 핵심 요약

-   SACK는 Selective Acknowledgement의 약자이다.
-   누적 ACK의 한계를 보완하기 위해 사용된다.
-   수신자는 이미 받은 데이터 구간을 송신자에게 알려준다.
-   송신자는 손실된 Segment만 재전송한다.
-   SACK는 TCP Header의 Options 필드를 사용한다.
-   대용량 전송에서 성능 향상 효과가 크다.
