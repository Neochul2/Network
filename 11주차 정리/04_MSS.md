# TCP/IP Advanced

# MSS (Maximum Segment Size)

------------------------------------------------------------------------

# 학습 목표

이번 장에서는 TCP에서 사용하는 **MSS(Maximum Segment Size)** 를
학습한다.

학습 후에는 다음 내용을 설명할 수 있어야 한다.

-   MSS란 무엇인가?
-   MTU와 MSS의 차이
-   MSS가 필요한 이유
-   기본 MSS가 536Byte인 이유
-   Fragmentation과 MSS의 관계

------------------------------------------------------------------------

# 1. MSS란?

MSS(Maximum Segment Size)는 **TCP 세그먼트의 Payload(Data)가 가질 수
있는 최대 크기**를 의미한다.

> **TCP Header는 MSS에 포함되지 않는다.**

많은 학생들이 MSS를 세그먼트 전체 크기로 오해하지만, MSS는
**Payload(Data)의 최대 크기만 의미**한다.

------------------------------------------------------------------------

# 2. 교수님 강조

교수님께서 가장 많이 강조하신 내용이다.

> "MSS는 Header가 아니라 Data의 크기이다."

시험에서도 자주 출제된다.

------------------------------------------------------------------------

# 3. MSS와 TCP Segment

예를 들어

``` text
TCP Header : 20Byte
Payload(Data) : 1460Byte
```

이라면

``` text
TCP Segment 전체 = 1480Byte
```

하지만

``` text
MSS = 1460Byte
```

이다.

------------------------------------------------------------------------

# 4. 왜 MSS가 필요한가?

TCP는 가능한 많은 데이터를 한 번에 전송하고 싶지만, 너무 큰 세그먼트는
IP 계층에서 **Fragmentation(단편화)** 을 발생시킨다.

Fragmentation이 발생하면

-   성능 저하
-   재조립 필요
-   패킷 손실 시 재전송 증가

등의 문제가 발생한다.

따라서 TCP는 MTU보다 작은 적절한 크기의 Payload를 전송하며, 이 기준이
MSS이다.

------------------------------------------------------------------------

# 5. MSS와 MTU의 차이

## MTU

Maximum Transmission Unit

네트워크에서 한 번에 전송 가능한 **IP Packet의 최대 크기**를 의미한다.

## MSS

Maximum Segment Size

TCP Payload(Data)의 최대 크기이다.

### 관계

``` text
MTU
=
IP Header
+
TCP Header
+
TCP Payload(Data)
```

따라서

``` text
MSS
=
MTU
-
IP Header
-
TCP Header
```

------------------------------------------------------------------------

# 6. Ethernet에서의 MSS

일반적인 Ethernet 환경에서는

-   MTU : 1500Byte
-   IPv4 Header : 20Byte
-   TCP Header : 20Byte

이므로

``` text
1500 - 20 - 20 = 1460Byte
```

따라서 일반적인 Ethernet에서는 **MSS = 1460Byte**를 사용한다.

------------------------------------------------------------------------

# 7. 기본 MSS 536Byte

초기 TCP에서는 IPv4의 최소 MTU를 **576Byte**로 가정하였다.

``` text
576 - 20(IP) - 20(TCP) = 536Byte
```

따라서 RFC에서 기본 MSS 값으로 **536Byte**가 사용되었다.

> 현재 Ethernet 환경에서는 보통 1460Byte가 사용되지만, **536Byte는
> 역사적인 기본 MSS 값**으로 시험에 자주 출제된다.

------------------------------------------------------------------------

# 8. MSS가 너무 작으면?

예를 들어

``` text
TCP Header : 20Byte
IP Header : 20Byte
Payload : 40Byte
```

Payload보다 Header 비율이 커져 대역폭을 비효율적으로 사용하게 된다.

------------------------------------------------------------------------

# 9. MSS가 너무 크면?

예를 들어 MSS를 3000Byte로 설정하면 Ethernet MTU(1500Byte)를 초과하여 IP
계층에서 Fragmentation이 발생한다.

문제점

-   성능 저하
-   IP 단편화 발생
-   재조립 필요
-   패킷 손실 시 재전송 증가

------------------------------------------------------------------------

# 10. Fragmentation과 MSS

``` text
MSS가 적절함
      │
      ▼
Fragmentation 없음
      │
      ▼
효율적인 전송
```

반대로

``` text
MSS가 너무 큼
      │
      ▼
Fragmentation 발생
      │
      ▼
성능 저하
```

------------------------------------------------------------------------

# 11. 교수님 수업 포인트

-   MSS는 Payload(Data)의 최대 크기이다.
-   TCP Header는 포함하지 않는다.
-   기본 MSS는 536Byte이다.
-   MTU와 혼동하지 말 것.

------------------------------------------------------------------------

# 12. 실무에서는?

Wireshark에서는 **TCP SYN 패킷의 TCP Options**에서 MSS 값을 확인할 수
있다.

예)

``` text
MSS = 1460
```

클라이언트와 서버는 **3-Way Handshake** 과정에서 MSS 옵션을 교환하며,
일반적으로 양쪽이 광고한 값 중 작은 값을 기준으로 통신한다.

------------------------------------------------------------------------

# 시험 포인트

-   MSS는 Payload(Data)의 최대 크기이다.
-   Header는 포함되지 않는다.
-   MSS = MTU - IP Header - TCP Header
-   역사적인 기본 MSS = 536Byte
-   일반적인 Ethernet MSS = 1460Byte

------------------------------------------------------------------------

# 암기법

``` text
MSS = Payload(Data)만!
```

2.  MTU와 MSS의 차이는?
3.  기본 MSS가 536Byte인 이유는?
4.  Ethernet에서 MSS가 1460Byte인 이유는?
5.  MSS와 Fragmentation의 관계는?

------------------------------------------------------------------------

# 핵심 요약

-   MSS는 TCP Payload(Data)의 최대 크기이다.
-   TCP Header는 MSS에 포함되지 않는다.
-   MSS는 MTU를 기준으로 결정된다.
-   Ethernet에서는 일반적으로 1460Byte를 사용한다.
-   536Byte는 역사적인 기본 MSS 값이다.
-   MSS는 IP Fragmentation을 줄이기 위해 사용된다.
